# GLOBAL PROJECT CONTEXT: membership-hub
## 1. Executive Summary & Tech Stack Blueprint
The membership-hub project is a comprehensive web and mobile application designed to manage and facilitate learning centers. The tech stack consists of Java 17, Quarkus, Kafka, Postgres, and Docker, with deployment on Google Cloud Platform (GCP) and Google Kubernetes Engine (GKE). The application will support authentication via email, password, Firebase, Google, and Facebook, and will have a role-based access control system with five distinct roles: System Admin, Admin, Manager, Teacher, and Student.

## 2. Global Guardrails & Enterprise Compliance Standards
To ensure the project's success and adherence to industry standards, the following global guardrails and enterprise compliance standards will be implemented:
* Data encryption and secure storage
* Access control and authentication mechanisms
* Regular security audits and penetration testing
* Compliance with relevant regulations, such as GDPR and CCPA
* Adherence to industry-recognized coding standards and best practices
* Continuous integration and continuous deployment (CI/CD) pipeline
* Monitoring and logging mechanisms for performance and error tracking

## 3. Standardized Sub-Agent Persona Definitions
The following sub-agent personas will be defined to ensure clear roles and responsibilities:
* **Manager**: Responsible for managing the development team, ensuring timely completion of tasks, and coordinating with stakeholders.
* **Coder**: Responsible for writing high-quality code, adhering to coding standards, and participating in code reviews.
* **Tester**: Responsible for testing the application, identifying bugs, and ensuring that the application meets the required standards.
* **Reviewer**: Responsible for reviewing code, providing feedback, and ensuring that the code meets the required standards.
* **DevOps**: Responsible for ensuring the smooth operation of the application, managing deployments, and monitoring performance.

## 4. Multi-Phase Segmentation Strategy Overview
The project will be segmented into exactly 5 phases, each with a duration of 1-7 days. The phases will be designed to ensure that all requirements are met, and the application is fully functional and tested.
* **Phase 1: Planning and Setup** (Days 1-2): Define project scope, create a detailed project plan, set up the development environment, and initialize the code repository.
* **Phase 2: Backend Development** (Days 3-5): Develop the backend API using Java 17, Quarkus, and Postgres, and implement authentication and authorization mechanisms.
* **Phase 3: Frontend Development** (Days 6-10): Develop the web and mobile frontend using Next.js, implement responsive design, and integrate with the backend API.
* **Phase 4: Testing and Quality Assurance** (Days 11-14): Conduct unit testing, integration testing, and UI testing, and ensure that the application meets the required standards.
* **Phase 5: Deployment and Monitoring** (Days 15-21): Deploy the application on GCP and GKE, set up monitoring and logging mechanisms, and ensure that the application is running smoothly.
